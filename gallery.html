삼성 디스플레이

프로젝트1

Project Description

  • 개발 기간 : 2022.04.06~2022.04.22

  • 참여 : 김기태, 이규호

  • Language : Java, JSP, HTML, CSS, Javascript, jQuery

  • IDE : eclipse 2021-09 (4.21.0)

  • RDBMS : MariaDB

  • Server&WAS : Tomcat 9.0

  • Biz Model : no MVC -> jsp

  • Version Control : Git/GitHub

프로젝트 개요

데스크탑 기준으로 기본 유틸리티는 Java로 구현하고, jsp 페이지에서 직접 데이터베이스를 접근하고, 필요한 데이터를 매핑하여 가져와 처리하였으며, jsp에서는 표현식과 스크립트릿, EL, JSTL 등을 활용하였으며, 기본 기능인 회원 로그인, 가입, 로그아웃, 게시판 글 목록, 게시글 보기, 게시판 글쓰기, 게시글 삭제 등의 기능을 구현하였음.

순수한 jsp만을 활용하여 웹 애플리케이션이라기 보다는 웹 사이트에 가깝게 제작되었으며, 기업형 사이트에 맞도록 기본 기능만 탑재하였고, 초기 프로젝트로서 다소 아쉬운 점도 많지만, 웹에서의 요청과 응답, 세션 처리, 웹의 구성과 처리 등을 숙달하도록 하였음.

여수 문화관광

프로젝트2

Project Description

  • 개발 기간 : 2022.04.23~2022.05.11

  • 참여 : 김기태, 김성민

  • Language : Java, JSP, Servlet, HTML, CSS, Javascript, jQuery, Ajax

  • IDE : eclipse 2021-09 (4.21.0)

  • RDBMS : Oracle XE 11g

  • Server&WAS : Tomcat 9.0

  • Biz Model : MVC Level 1 (Contoller->Model->View)

  • Version Control : Git/GitHub

프로젝트 개요

Controller와 데이터베이스 접근 및 처리를 위한 Model을 활용하여 필요한 데이터 처리 및 요청에 따라 서비스를 처리하게 하였으며, VO와 Controller, Model 등은 Java로 작성되었으며, View 부분은 jsp의 표현식과 스크립트릿, EL, JSTL 등을 활용하여, 기본 기능인 회원 로그인, 가입, 로그아웃, 게시판 글 목록, 게시글 보기, 게시판 글쓰기, 게시글 삭제 등의 기능과 관리자의 회원목록, 회원정보변경 등 을 구현하였음.

뷰(view)인 jsp에서는 헤더, 콘텐츠, 푸터 부분을 각 각 모듈화하여 인클루드하였으며, Controller에서 뷰를 통하여 서비스를 요청받아 Model에서 DB에 접근하여 요청 정보를 받아 다가 다시 Controller로 보낸 후 View 에 응답 데이터를 출력하는 구조로 진행하였음.

뷰(view)인 jsp에서는 헤더, 콘텐츠, 푸터 부분을 각 각 모듈화하여 인클루드하였으며, Controller에서 View를 통하여 서비스를 요청받아 Model과 DAO에서 DB에 접근하여 요청 정보를 받아 다가 다시 Controller에서 View로 응답 데이터를 처리하게 하는 방식으로 진행하였음.

하나 이비인후과 병원

프로젝트4

Project Description

  • 개발 기간 : 2022.06.08~2022.06.30

  • 참여 : 김기태, 임소희

  • Language : Java, JSP, Servlet, MyBatis, DI, IoC, HTML, CSS, Javascript, jQuery, Ajax, jUnit

  • IDE : Spring Studio Suite 3.9.12 Release

  • RDBMS : Oracle XE 11g

  • Server&WAS : Tomcat 9.0

  • Biz Model & Build : Spring Maven MVC Pattern

  • Version Control : Git/bitbucket

  • Dependensies : lombok, jackson, gson, spring-security, cos, javax.mail, validation, hibernate

프로젝트 개요

Controller 어노테이션을 활용하여 Controller를 만들고, 필요한 Service를 주입하여 각 RequestMapping 어노테이션에서 요청을 파라미터와 함께 접수하고, 요청 정보를 Service 에 전달하여 Repository 어노테이션에 의해 정의된 DAO로 파라미터로 받아 다가 Mapper인 MyBatis 에서 DB에 간접 접근하여 응답 정보를 받아 View 까지 전달하도록 하였고, View 부분은 jsp의 EL, JSTL 등을 활용하여, 기본 기능인 회원 목록, 로그인, 회원가입, 회원정보수정, 비밀번호 암호화, 게시판 목록, 게시글 보기, 응답형 게시판, 자료실 게시판, 게시글 삭제, 게시글 수정, 온라인 상담, 이메일 보내기 등의 기능을 구현하였음.

구현 Logic View -> Controller -> Service -> DAO -> MyBatis -> DB
